Для запуска тестов достаточно запустить `make.do` скрипт

Ошибки выводятся в формате: `{time} ns: {error}`. Например `{time} ns: {name} expected {ref_val} but got {dut_val}`

- `ast_endofpacket_o`: не выставляется в единицу при размере пакета равным `1` (сообщение выводится `65 ns`, но сама ошибка `35 ns`) (пакет с `channel = 82`)
- `ast_valid_i`:       сигнал игнорируется (когда `generator` отправляет `gibberish`) (сообщение выводится начиная с `684925 ns`)
- `ast_ready_i`:       если сигнал редко поднимать в `1`, модуль пропускает некоторые пакеты. Чтобы это проверить, в файле [`ast_we_generator.sv`](./tb/ast_we_generator.sv#L10) нужно изменить параметр `SRC_READY_CHANCE` на малое значение (`1` - `20`)
- `ast_data_o`:        на протяжении всего теста можно заметить, что модуль выдаёт неверные данные, либо вообще пакеты некорректной длины (начиная с `135 ns`)
