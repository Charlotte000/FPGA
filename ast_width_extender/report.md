Для запуска тестов достаточно запустить `make.do` скрипт

Ошибки выводятся в формате: `{time} ns: {error}`. Например `{time} ns: {name} expected {ref_val} but got {dut_val}`

Для наглядности и чтобы не загромождать консоль вывода я рекомендую проводить один тест за раз. Для этого в файле [ast_we_tb.sv](./tb/ast_we_tb.sv#102) необходимо закомментировать все тесты кроме одного желаемого. Далее все тайминги будут приведены имеено в таком формате.

- `send_different_lengths`:
    - (1) `ast_endofpacket_o` не выставляется в единицу при размере пакета меньшим `DATA_IN_W` бит (`8364735 ns`, пакеты с `channel` = `503`, `459`, `677`)
    - (2) `ast_data_o` на протяжении всего теста можно заметить, что модуль выдаёт пакеты некорректной длины (ошибки типа `packet size expected x but got y (channel=z)`)
    - (3) `ast_ready_i` если сигнал редко поднимать в `1`, модуль пропускает некоторые пакеты. Чтобы это проверить, в файле [ast_we_tb.sv](./tb/ast_we_tb.sv#L16) нужно изменить значение параметра `SNK_READY_CHANCE` на малое значение (`1` - `20`) (ошибки типа `unexpected ast_startofpacket_o`)
- `send_gibberish`:
    - (4) `ast_valid_i` сигнал игнорируется. в этом тесте `valid_i` сигнал стоит в `0`, но модуль всё равно пытается обрабатывать данные (ошибки типа `unexpected ast_startofpacket_o`)
- `send_zero_empty` и `send_non_zero_empty`:
    - (2) все пакеты некорректной длины являются пакетами, в которых `ast_empty_i = 0`. из всех пакетов, в которых `ast_empty_i = 0`, приблизительно `25/100 = 25%` пакетов передаются некорректно
